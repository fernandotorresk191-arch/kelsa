# 🔧 Backoffice Kelsa — Руководство администратора

**Последнее обновление:** Апрель 2026

## Что такое Backoffice?

Backoffice (бэкофис) — скрытая от обычных пользователей часть приложения для менеджеров и администраторов. Здесь вы управляете заказами, товарами, закупками, курьерами, зонами доставки и аналитикой.

**Доступ:** `https://bo.gokelsa.ru` или `/admin`

---

## 🚀 Начало работы

### 1. Вход в систему

Перейдите на страницу логина:
```
https://bo.gokelsa.ru/login
```

Введите свои учетные данные (email и пароль, выданные администратором).

### 2. Главный экран

После входа вы увидите боковое меню с 15 разделами:

**Основное:**
- Дашборд — основная статистика
- Заказы — управление заказами + чат
- Каталог — категории и подкатегории
- Товары — управление каталогом
- Баннеры — акции и промо

**Склад:**
- Закупки — приёмка товаров, создание партий
- Просрочка — отслеживание сроков годности, списание

**Отчёты:**
- Аналитика — графики, статистика, KPI
- Клиенты — база покупателей, LTV, история заказов
- Клиенты — база покупателей, LTV, история заказов

**Доставка:**
- Курьеры — управление курьерами
- Зоны доставки — тарифы по населённым пунктам

**Управление:**
- Дарксторы — управление складами-магазинами (superadmin)

**Система:**
- Пользователи — админы и менеджеры
- Сервер — системная информация

### 3. Переключение дарксторов

Если вам назначено несколько дарксторов, в шапке появится переключатель. Все данные (заказы, товары, курьеры) фильтруются по выбранному даркстору.

### Просмотр списка заказов

Перейдите в раздел **Заказы**.

Здесь вы видите:

### Фильтрация по статусу

Используйте выпадающий список для фильтрации заказов по статусам.

### Просмотр деталей заказа

Кликните на **"Просмотр →"** для открытия деталей заказа.

#### На странице заказа:

**Информация о заказе:**
- Клиент, телефон, адрес доставки
- Комментарий клиента
- Населённый пункт, даркстор

**Товары в заказе:**
Таблица: название, цена, количество, сумма

**Экономика заказа:**
- Subtotal, себестоимость, тариф курьера, доставка, прибыль

- Выберите доступного курьера из списка

**Чат с клиентом:**

- Геолокация

**Печать:**

- Лист сборки — для сборщика

## 📂 Каталог (Категории)

Перейдите в раздел **Каталог**.

### Иерархия категорий
Категории двухуровневые: категория → подкатегории.

### Управление категориями
- Создание новых категорий и подкатегорий
- Редактирование: название, slug, изображение, сортировка
- Наценка (markupPercent) — автоматически применяется к товарам
- Активация/отключение

### Включение/отключение категорий для даркстора (DarkstoreCategory)

Категории являются **глобальными** — они общие для всех дарксторов. Однако каждый даркстор может включать/отключать отдельные категории через тогл-переключатель в колонке **«В дарксторе»**.

- Зелёный тогл = категория включена для текущего даркстора
- Серый тогл = категория отключена, клиенты не видят её на сайте
- При создании новой категории она автоматически включается во всех дарксторах

## 🛍️ Управление товарами

### Просмотр списка товаров

Перейдите в раздел **Товары**.

Таблица показывает:
- Изображение, название
- Цену, старую цену
- Остаток на складе
- Ячейку хранения
- Статус (активный/отключенный)

### Добавление нового товара

1. Кликните **"+ Добавить товар"**
2. Заполните: название, slug, описание, штрихкод, вес
3. Загрузите изображение
4. Выберите категорию и подкатегорию
5. Товар появится в каталоге

### Цены и остатки

Цены и остатки управляются через **DarkstoreProduct** — привязку товара к даркстору:
- Цена продажи, старая цена (для скидок)
- Закупочная цена
- Остаток на складе
- Номер ячейки хранения
- **Максимум на заказ** (`maxPerOrder`, по умолчанию 10) — лимит штук одного товара в одном заказе

> **Резервирование:** При создании заказа остаток товара уменьшается атомарно. Если два клиента одновременно оформляют заказ на последний товар, только один из них успешно оформит — второй увидит сообщение «Недостаточно товара на складе».

### Отключение товара

Переключите опцию **"Товар активен"** и сохраните. Товар не будет удален, но не будет отображаться на сайте.

## 📥 Закупки и партии

### Создание закупки

Перейдите в раздел **Закупки** → **"+ Новая закупка"**.

1. Укажите поставщика и примечания
2. Добавьте позиции:
   - Товар, количество, закупочная цена
   - Ячейка хранения, срок годности
   - Наценка (или автоматически из категории)
3. Система создаст **Партии** (Batch) с уникальными кодами

### Как работают партии (FIFO)

Каждая позиция закупки становится **партией** со статусом:
- `ACTIVE` — партия активна, товары продаются
- `SOLD_OUT` — партия распродана
- `EXPIRED` — срок годности истек
- `WRITTEN_OFF` — списана

**Цена на витрине** определяется первой активной партией по FIFO (ближайший срок годности).

При **доставке заказа** система автоматически списывает товары по FIFO.

## � Клиенты

Перейдите в раздел **Клиенты** (секция «Отчёты»).

Раздел показывает всю базу покупателей сети. Если выбран конкретный даркстор — отображаются только клиенты, размещавшие заказы в нём.

### Список клиентов

**Сводные плашки (верхняя строка):**
- **Всего клиентов** — общее число зарегистрированных покупателей
- **Заказов** — суммарное число заказов на текущем экране
- **Выручка** — суммарные траты клиентов текущего экрана (доставленные заказы)
- **Средний чек** — выручка / заказы

**Поиск:** по имени, телефону, email или логину.

**Сортировка:**
| Режим | Описание |
|---|---|
| Последний заказ | Сначала недавно заказывавшие |
| По выручке | Лучшие покупатели по LTV |
| По числу заказов | Самые частые покупатели |
| Дата регистрации | Хронологически |

**Таблица клиентов:**
- Аватар с инициалами (цвет привязан к id клиента)
- Имя + email
- Телефон
- Населённый пункт (settlement)
- Всего заказов / доставлено + конверсия (%)
- Суммарная выручка (только доставленные)
- Дата последнего заказа
- Бейдж активности:
  - 🟢 **≤ 7 дней** — активный клиент
  - 🔵 **≤ 30 дней** — недавно заказывал
  - 🟡 **≤ 90 дней** — «засыпает»
  - ⚪ **N дн.** — неактивен

---

### Профиль клиента

Нажмите **«Профиль →»** для перехода на детальную страницу клиента.

#### Hero-карточка
- Имя, контакты, адрес, населённый пункт
- Дата регистрации и число дней в базе
- **Сегмент клиента:**
  - 🥈 **Новичок** — менее 2 доставленных заказов
  - 🔵 **Returning** — 2–4 доставленных заказа
  - 🟣 **Постоянный** — 5–9 заказов
  - 🥇 **VIP** — 10+ заказов и сумма ≥ 10 000 ₽
- **Бейдж активности** (по дням с последнего заказа)

#### Метрики
| Карточка | Что показывает |
|---|---|
| Всего заказов | Все заказы (любой статус) |
| Доставлено (конверсия%) | Доставленные + % от всех + число отменённых |
| Суммарные траты + доставка | LTV клиента по доставленным заказам |
| Средний чек | avgOrderValue + дата первого заказа |

#### Статусы заказов
График-плашки: каждый статус с количеством и прогресс-баром.

#### Любимые товары
Топ-5 товаров по штукам из **доставленных** заказов:
- Место, название, число заказов, кол-во штук, суммарные траты

#### История заказов
Последние 30 заказов: номер (кликабельный → страница заказа), статус, дата, н.п., даркстор, состав (первые 2 позиции), сумма.

## 🚚 Курьеры

Перейдите в раздел **Курьеры**.

### Управление курьерами
- Создание: ФИО, логин, пароль, телефон, автомобиль
- Статусы: OFF_DUTY, AVAILABLE, ACCEPTED, DELIVERING

### Курьерское приложение
Курьеры используют отдельное PWA-приложение по адресу `/courier`, где они:
- Получают push-уведомления о новых заказах
- Принимают/отклоняют заказы
- Отправляют GPS-координаты
- Обновляют статусы

## 🐛 Устранение неполадок

### Не могу войти
1. Проверьте email и пароль
2. Попросите администратора проверить, активен ли аккаунт
3. Очистите кэш браузера

### Не вижу данных
1. Проверьте, выбран ли правильный даркстор в шапке
2. Суперадмин без выбранного даркстора видит все данные

### В разделе Клиенты нет клиентов
1. Если выбран даркстор — раздел показывает только тех, кто делал заказы именно в этом дарксторе
2. Суперадмин без выбранного даркстора видит всех клиентов сети

### Товар не появляется на сайте
1. Проверьте isActive = true
2. Проверьте, что есть DarkstoreProduct с ценой и stock > 0
3. Проверьте привязку к категории
