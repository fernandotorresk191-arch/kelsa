## 👤 Пошаговое руководство пользователя

### Сценарий 1: Новый пользователь (Регистрация)

1. **Открывается корзина**
   - Вид: Жёлтый блок с сообщением "Чтобы оформить заказ, зарегистрируйтесь или войдите."
   - Кнопка: "Зарегистрироваться" (серая)
   - Кнопка оформления: "Войти, чтобы оформить" (серая, неактивна)

2. **Пользователь кликает "Зарегистрироваться"**
   - Открывается модальное окно регистрации
   - Вкладка "Регистрация" по умолчанию активна

3. **Вводит данные регистрации**
   - Логин: `ivan_petrov`
   - Пароль: `SecurePass123`
   - Имя: `Иван Петров` ← **Важно: имя будет использоваться в приветствии**
   - Телефон: `+7 999 123-45-67`
   - Email: `ivan@example.com`
   - Адрес: `ул. Ленина, д. 5, кв. 10`
   - Село: `Калиновская`
   - Нажимает "Зарегистрироваться"

4. **После успешной регистрации**
   - Модальное окно закрывается автоматически
   - В корзине появляется **зелёный блок**:
     ```
     Здравствуйте, Иван Петров! Мы готовы принять ваш заказ!
     ```
   - Жёлтый блок исчезает
   - Кнопка становится зелёной: "Оформить заказ"

5. **Оформление заказа**
   - Указывает контактные данные (они могут быть предзаполнены)
   - Кнопка "Оформить заказ" активна (зелёная, кликабельна)
   - Нажимает "Оформить заказ"

6. **Успешное оформление**
   - Появляется **зелёный блок успеха**:
     ```
     ✓ Заказ №12345 успешно оформлен!
     
     Его статус вы можете отслеживать в личном кабинете. 
     Мы свяжемся с вами для подтверждения.
     
     Сумма заказа: 5 490 ₽
     [Продолжить покупки]
     ```

---

### Сценарий 2: Существующий пользователь (Вход)

1. **Открывается корзина**
   - Жёлтый блок с предложением регистрации/входа

2. **Пользователь кликает "Зарегистрироваться"**
   - Открывается модальное окно регистрации

3. **Переключается на вкладку "Вход"**
   - Вводит логин и пароль
   - Нажимает "Войти"

4. **После успешного входа**
   - Модальное окно закрывается
   - Появляется приветствие с именем
   - Кнопка оформления становится зелёной

## 🧪 Как протестировать

### Требования
- Docker и Docker Compose
- Node.js 18+
- PostgreSQL (или запущен в контейнере)

### Шаг 1: Запустите бэкэнд

```bash
cd backend/api

# Установите зависимости (если не установлены)
npm install

# Запустите миграции БД
npx prisma migrate dev

# Запустите сервер
npm run start:dev
```

Бэкэнд должен быть доступен на `http://localhost:3333`

### Шаг 2: Запустите фронтэнд

```bash
cd frontend

# Установите зависимости (если не установлены)
```

## ❓ FAQ

**Q: Что произойдёт с существующими пользователями без имени?**  
A: У всех пользователей в БД уже есть поле name, установленное при регистрации.

**Q: Почему кнопка становится зелёной?**  
A: Для визуального контраста и подтверждения готовности к оформлению заказа.

**Q: Как получить номер заказа?**  
A: Номер приходит в ответе API и отображается в блоке успеха.

**Q: Где отслеживать статус заказа?**  
A: В личном кабинете (страница `/account`).

## 🆘 Решение проблем

### Проблема: Имя пользователя не показывается
**Решение:**
1. Проверьте, что API возвращает поле `name` в ответе
2. Проверьте консоль браузера на ошибки
3. Убедитесь, что пользователь авторизован

### Проблема: Кнопка остаётся серой
**Решение:**
1. Проверьте, что пользователь авторизован (user !== null)
2. Проверьте консоль браузера на ошибки React
3. Попробуйте перезагрузить страницу

### Проблема: Модальное окно не закрывается
**Решение:**
1. Проверьте консоль на ошибки авторизации
2. Убедитесь, что API доступен
3. Проверьте CORS настройки если используется другой домен
