# Экономика проекта Kelsa

Документ описывает полную схему формирования прибыли, ценообразования, логистики доставки и финансовой аналитики.

---

## 1. Жизненный цикл товара: от закупки до продажи

### 1.1. Закупка и формирование партий

Менеджер создаёт **Закупку** (`Purchase`) с набором позиций. Для каждой позиции:

```
Входные данные:
  productId        — какой товар
  quantity         — количество
  purchasePrice    — закупочная цена (₽)
  cellNumber       — ячейка хранения
  markupPercent    — наценка (%, опционально)
  expiryDate       — срок годности (опционально)
```

Система создаёт **Партию** (`Batch`) с уникальным кодом формата `ЯЧЕЙКА ЗАКУПКА ТОВАР` (напр. `24 00001 18`) и рассчитывает цену продажи:

```
sellingPrice = purchasePrice × (1 + markupPercent / 100)   // округляется вниз
```

**Откуда берётся наценка** (приоритет):
1. Указана явно в позиции закупки
2. Наценка подкатегории товара (`subcategory.markupPercent`)
3. Наценка категории товара (`category.markupPercent`)
4. 0% (без наценки)

После создания партии:
- `Product.stock` увеличивается на `quantity`
- `Product.purchasePrice` обновляется до закупочной цены последней партии
- Пересчитывается `Product.price` (см. ниже)

### 1.2. Ценообразование по FIFO

Цена товара на витрине определяется **активной партией** по правилу FIFO:

1. Выбирается первая партия со статусом `ACTIVE` и `remainingQty > 0`
2. Сортировка: `expiryDate ASC` → `createdAt ASC` (сначала ближайший срок годности)
3. Если у партии есть скидка за истекающий срок:

```
effectivePrice = sellingPrice × (1 − discountPercent / 100)
```

4. Устанавливается:
   - `Product.price = effectivePrice` — текущая цена
   - `Product.oldPrice = sellingPrice` — зачёркнутая цена (если есть скидка)

### 1.3. Списание остатков при доставке

При **доставке** заказа (статус `DELIVERED`) система автоматически списывает товары из партий по FIFO:

1. Для каждого товара в заказе уменьшается `Product.stock`
2. Находятся активные партии (`ACTIVE`, `remainingQty > 0`), отсортированные по `expiryDate ASC` → `createdAt ASC`
3. Из партий списывается нужное количество (`remainingQty -= qty`)
4. Если партия полностью распродана (`remainingQty = 0`), её статус меняется на `SOLD_OUT`
5. Пересчитывается цена товара на витрине по следующей активной партии (FIFO)

## 2. Зоны доставки

Доставка тарифицируется по **населённым пунктам**. Каждый населённый пункт — отдельная зона (`DeliveryZone`):

| Параметр | Описание |
|---|---|
| `settlement` | Код населённого пункта (enum) |
| `deliveryFee` | Тариф доставки в рублях (начисляется курьеру) |
| `freeDeliveryFrom` | Порог бесплатной доставки для клиента |
| `isActive` | Включена ли доставка |

**Населённые пункты:**
- Калиновская (`KALINOVSKAYA`)
- Новотерская (`NOVOTERSKAYA`)
- Левобережное (`LEVOBEREZHNOE`)
- Юбилейное (`YUBILEYNOE`)
- Новое-Солкушино (`NOVOE_SOLKUSHINO`)

У каждой зоны свой тариф и свой порог бесплатной доставки.

## 3. Формирование прибыли заказа

### 3.1. Когда считается прибыль

**Прибыль рассчитывается только после доставки** (статус `DELIVERED`). При создании заказа `profit = 0`. Это гарантирует, что:
- Отменённые и недоставленные заказы не искажают финансовую аналитику
- Прибыль отражает только реально выполненные заказы
- При отмене доставленного заказа прибыль обнуляется обратно

### 3.2. Исходные данные при оформлении

| Параметр | Источник |
|---|---|
| Цена товара (`price`) | `Product.price` — с FIFO-партии |
| Закупочная цена (`purchasePrice`) | `Product.purchasePrice` — последняя партия |
| Тариф зоны (`deliveryFee`) | `DeliveryZone.deliveryFee` — по населённому пункту |
| Порог бесплатной доставки | `DeliveryZone.freeDeliveryFrom` |

### 3.3. Формулы расчёта (при доставке)

```
subtotal     = Σ (item.amount)                       — сумма товаров (из позиций заказа)
purchaseCost = order.purchaseCost                     — себестоимость (сохранена при создании)
courierCost  = order.courierCost                      — тариф курьера (сохранён при создании)

                 ┌ 0,                       если subtotal ≥ freeDeliveryFrom
deliveryFee  =   │
                 └ DeliveryZone.deliveryFee, если subtotal < freeDeliveryFrom

totalAmount  = subtotal + deliveryFee                — итого к оплате клиентом
profit       = subtotal − purchaseCost − courierCost — чистая прибыль
```

### 3.4. Денежные потоки

```
КЛИЕНТ ПЛАТИТ: totalAmount = subtotal + deliveryFee
┌───────────────────────────────────────────────────┐
│                                                   │
│   subtotal (за товары)  +  deliveryFee (доставка) │
│                                                   │
└────────────┬──────────────────────┬───────────────┘
             │                      │
             ▼                      ▼
┌────────────────────────┐   Плата за доставку
│   ВЫРУЧКА = subtotal   │   (может быть 0 при
│                        │   бесплатной доставке)
├────────────────────────┤
│                        │
│  − purchaseCost  ──────┼──▶  Поставщику (себестоимость товаров)
│                        │
│  − courierCost   ──────┼──▶  Курьеру (тариф зоны, ВСЕГДА)
│                        │
├────────────────────────┤
│  = PROFIT              │     Чистая прибыль бизнеса
└────────────────────────┘
```

### 3.5. Ключевые принципы

- **Прибыль считается от subtotal** (суммы товаров), а не от totalAmount. Доставка, оплаченная клиентом, не участвует в расчёте прибыли — она компенсирует расход на курьера.
- **courierCost начисляется всегда**, даже если клиент не платит за доставку (при бесплатной доставке). Это расход бизнеса.
- **Прибыль может быть отрицательной**, если `purchaseCost + courierCost > subtotal`.
- **Тариф привязан к зоне** (населённому пункту), а не к конкретному курьеру.
- **Прибыль записывается только при DELIVERED** — до этого `profit = 0`.
- **При отмене доставленного заказа** — `profit` обнуляется обратно.

## 4. Пример расчёта

### Пример A: Заказ ниже порога бесплатной доставки

> 3 товара по 300 ₽ (закупка 200 ₽), Калиновская (тариф 150 ₽, бесплатно от 1500 ₽)

| Показатель | Расчёт | Значение |
|---|---|---|
| subtotal | 3 × 300 | **900 ₽** |
| purchaseCost | 3 × 200 | **600 ₽** |
| courierCost | тариф зоны | **150 ₽** |
| deliveryFee (клиенту) | 900 < 1500 → тариф | **150 ₽** |
| totalAmount | 900 + 150 | **1 050 ₽** |
| **profit** | 900 − 600 − 150 | **150 ₽** |

Клиент платит 1 050 ₽. Курьер получает 150 ₽. Прибыль бизнеса — 150 ₽.

### Пример B: Заказ выше порога (бесплатная доставка)

> 6 товаров по 300 ₽ (закупка 200 ₽), Калиновская (те же параметры)

| Показатель | Расчёт | Значение |
|---|---|---|
| subtotal | 6 × 300 | **1 800 ₽** |
| purchaseCost | 6 × 200 | **1 200 ₽** |
| courierCost | тариф зоны | **150 ₽** |
| deliveryFee (клиенту) | 1800 ≥ 1500 → 0 | **0 ₽** |
| totalAmount | 1800 + 0 | **1 800 ₽** |
| **profit** | 1800 − 1200 − 150 | **450 ₽** |

Клиент платит 1 800 ₽. Доставка бесплатна для клиента, но курьер всё равно получает 150 ₽ от бизнеса. Прибыль — 450 ₽.

### Пример C: Убыточный заказ

> 1 товар по 250 ₽ (закупка 200 ₽), зона с тарифом 200 ₽

| Показатель | Расчёт | Значение |
|---|---|---|
| subtotal | 1 × 250 | **250 ₽** |
| purchaseCost | 1 × 200 | **200 ₽** |
| courierCost | тариф зоны | **200 ₽** |
| deliveryFee (клиенту) | 250 < порог → 200 | **200 ₽** |
| totalAmount | 250 + 200 | **450 ₽** |
| **profit** | 250 − 200 − 200 | **−150 ₽** |

Клиент заплатил 450 ₽, но бизнес в убытке на 150 ₽. Маржа товара (50 ₽) не покрывает расходы на курьера (200 ₽).

## 5. Заработок курьеров

Курьер получает фиксированный тариф за каждый **доставленный** заказ (`status = DELIVERED`):

```
Заработок курьера = Σ courierCost по всем доставленным заказам
```

Тариф определяется зоной доставки заказа, а не параметрами курьера. Один курьер может доставлять в разные зоны с разными тарифами.

**Периоды расчёта:**
- За сегодня
- За неделю (с понедельника)
- За месяц (с 1 числа)
- Всего за всё время

## 6. Ценообразование: полная цепочка

```
Поставщик                     Склад                      Клиент
    │                           │                           │
    │  purchasePrice            │  sellingPrice              │  price (effectivePrice)
    │  (закупочная)             │  = purchasePrice           │  = sellingPrice
    │───────────────────────▶   │    × (1 + markup%)         │    × (1 − discount%)
    │                           │──────────────────────────▶ │
    │                           │                           │
    │                           │  Если есть скидка:        │  Итого клиенту:
    │                           │  oldPrice = sellingPrice  │  totalAmount = subtotal
    │                           │  price = effectivePrice   │    + deliveryFee
    │                           │                           │
```

**Наценка** (`markupPercent`) — процент от закупочной цены, формирующий цену продажи партии.

**Скидка** (`discountPercent`) — процент от цены продажи, применяемый для товаров с истекающим сроком годности.

## 7. Аналитика

> **Все финансовые показатели в аналитике рассчитываются только по заказам со статусом `DELIVERED`.**

### 7.1. Дашборд (общие показатели)

| Метрика | Формула |
|---|---|
| Всего заказов | `COUNT(orders)` |
| Доставлено заказов | `COUNT(orders WHERE status = DELIVERED)` |
| Общая выручка | `SUM(totalAmount) WHERE status = DELIVERED` |
| Выручка за сегодня | `SUM(totalAmount) WHERE status = DELIVERED AND createdAt ≥ начало_дня` |
| Себестоимость | `SUM(purchaseCost) WHERE status = DELIVERED` |
| Расходы на курьеров | `SUM(courierCost) WHERE status = DELIVERED` |
| Общая прибыль | `SUM(profit) WHERE status = DELIVERED` |
| Прибыль за сегодня | `SUM(profit) WHERE status = DELIVERED AND createdAt ≥ начало_дня` |
| Средний чек | `SUM(totalAmount) / COUNT(orders) WHERE status = DELIVERED` |

### 7.2. Аналитика выручки (по дням)

Группировка по дням **только по доставленным заказам**:
- **Выручка** (`revenue`) = `SUM(totalAmount)` за день
- **Себестоимость** (`purchaseCost`) = `SUM(purchaseCost)` за день
- **Курьеры** (`courierCost`) = `SUM(courierCost)` за день
- **Прибыль** (`profit`) = `SUM(profit)` за день

Доступные периоды: неделя, месяц, год.

### 7.3. Статистика заказов (по диапазону дат)

Группировка по дням:
- Количество заказов
- Выручка
- Прибыль
- Список заказов с деталями

### 7.4. Карточка заказа

Каждый заказ в админке показывает экономику:
- Выручка (totalAmount)
- Себестоимость (purchaseCost)
- Доставка курьеру (courierCost)
- Прибыль (profit) — зелёная если ≥ 0, красная если < 0

## 9. Формула маржинальности

```
Маржа валовая = (subtotal − purchaseCost) / subtotal × 100%
Маржа чистая  = profit / subtotal × 100%
                = (subtotal − purchaseCost − courierCost) / subtotal × 100%
```

**Рычаги управления прибылью:**
1. **Наценка** (`markupPercent`) — увеличивает цену продажи
2. **Порог бесплатной доставки** (`freeDeliveryFrom`) — стимулирует больший средний чек
3. **Тарифы по зонам** (`deliveryFee`) — управление расходами на логистику
4. **Скидки на истекающие партии** (`discountPercent`) — минимизация списаний за счёт снижения маржи
